## Principe Fondamental

Toute recherche d'information dans Y-OS suit un protocole à deux niveaux distincts. Ce principe s'applique à toute situation où Manus cherche des sources d'information — monitoring récurrent, recherche ponctuelle, ou veille thématique.


## Niveau 1 : Meta-Monitoring (Trouver les Sources)

Avant de chercher des ressources, l'agent cherche d'abord où chercher. L'objectif est de découvrir de nouveaux portails, forums, répertoires GitHub, subreddits, guides, ou bases de données dédiés au domaine cible. Ces sources sont évaluées sur leur densité de contenu, leur engagement communautaire, et leur pertinence. Une fois validées, elles sont ajoutées à une Approved Source List persistante dans Notion.

Fréquence recommandée : 1x/mois — les sources évoluent lentement.


## Niveau 2 : Extraction (Trouver les Ressources)

Une fois la Source List établie, l'agent scrape systématiquement chaque source validée pour en extraire les meilleures ressources (skills, scripts, outils, MCPs, etc.). Il applique des filtres stricts : étoiles GitHub, volume de téléchargements, commentaires, réputation communautaire, et mise en avant officielle.

Fréquence recommandée : 2x/mois — les ressources évoluent rapidement.


## Règle d'Application Universelle

Ce protocole s'applique à tout domaine de recherche dans Y-OS : veille SuperManus, recherche de datasets, veille sectorielle, toute recherche multi-sources récurrente.
