## Lessons Learned

Worked well:

- Web-based approach provided cross-platform compatibility
- International Typographic Style created distinctive, professional appearance
- Progress indicators significantly improved user experience
- Comprehensive testing prevented major deployment issues
Failed / suboptimal:

- Initial routing configuration caused 404 errors
- Python 3.13 compatibility issues required downgrade to 3.11
- API error handling needed improvement for better user feedback
Discoveries:

- HTML error pages returned as JSON caused parsing errors
- Visual feedback during long operations is crucial for user retention
- Clean environment setup prevents module conflicts

## Challenges & Blockers

- Python version compatibility issues
- API endpoint routing errors
- JSON parsing errors from HTML responses
- Long script execution times without user feedback
