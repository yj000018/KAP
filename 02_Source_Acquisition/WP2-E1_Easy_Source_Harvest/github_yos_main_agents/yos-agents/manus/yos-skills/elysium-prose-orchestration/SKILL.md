# ELYSIUM Prose Orchestration

This skill defines the canonical multi-LLM orchestration workflow for drafting ELYSIUM book modules.

## Overview

Manus is the **orchestrator**, not the author. Prose is generated by the **Claude API** and reviewed by the **ChatGPT API**. The Founder acts as co-pilot in FSD Controlled Mode.

## Workflow: Module Loop

For each module inside an approved scope:

1. **Preflight**: Verify gates (validation clean, API keys safe, required source files exist).
2. **Read context**: Read previous module draft, current writing brief, and current X-Ray.
3. **Build context pack**: Create `BOOK/_fcs/context_packs/PHASE_XXX_MOD_PROSE_CONTEXT_PACK.md`.
4. **Call Claude API**: Generate prose via `claude-opus-4-5`. Save to `BOOK/_fcs/api_outputs/PHASE_XXX_MOD_CLAUDE_RAW.md`.
5. **Call ChatGPT API**: Review prose via `gpt-4o-2024-08-06`. Save to `BOOK/_fcs/reviews/PHASE_XXX_MOD_CHATGPT_REVIEW.md`.
6. **Handle Review Decision**:
   - **If PASS**: Proceed to step 7.
   - **If REVISE**: Call Claude API for one revision based on instructions. Call ChatGPT API for final review. If PASS, proceed. If REVISE/FAIL, STOP.
   - **If FAIL**: STOP and request Founder review.
7. **Save DRAFT_0**: Save approved prose to `BOOK/manuscript/.../drafts/MOD_DRAFT_0.md` with proper frontmatter.
8. **Update Registries**: Update `opening_writing_brief_registry.yaml` and `opening_prose_draft_registry.yaml`.
9. **Validate**: Run `scripts/validate.py BOOK` and `scripts/status.py BOOK`.
10. **Report**: Create `BOOK/_fcs/reports/PHASE_XXX_MOD_PROSE_REPORT.md` and Founder review request in `BOOK/_fcs/action_requests/active/`.
11. **Commit/Tag/Push**: Commit changes. Tag if appropriate for the scope. Push to GitHub.
12. **Mem0**: Persist completion fact to Mem0.

## FSD Controlled Mode Constraints

- **Scope**: Operate only within the explicitly approved scope (e.g., MOV-II OPN-004 to OPN-006).
- **Stop Gates**: Stop immediately on review failure, validation error, tone drift, critical module (OPN-008, OPN-012), or scope end.
- **No WYS**: Use yOS only.
- **No Manus Prose**: Manus must not write the book prose.
