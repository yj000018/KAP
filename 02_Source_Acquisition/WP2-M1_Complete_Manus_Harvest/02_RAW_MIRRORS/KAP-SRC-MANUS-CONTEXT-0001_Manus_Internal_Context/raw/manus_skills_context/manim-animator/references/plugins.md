# Manim Plugins (on-demand)

Consult this file in Stage 3 (scene planning) whenever any trigger keyword below appears in the user prompt or in `scenes.md`. Plugins are **never preinstalled**; if a scene needs one, install it lazily at the top of the generated `video.py`. Never add plugins to `scripts/setup.sh`.

## 1. Trigger table

For each row, if the trigger matches, record the plugin in the scene's "technical notes" field as `plugin: <name>` before writing code. Only the listed plugin is allowed for that job; do not substitute a sibling fork.

| Trigger (prompt keywords or scene contents) | Plugin | Adds | Constraints |
|---|---|---|---|
| `MathTex`/`Tex` in many scenes AND sandbox has no TeX Live | `manim-onlinetex` | Renders LaTeX via an online service, removing the texlive dependency | Single run should not exceed ~50 MathTex calls (server rate limits). On failure, fall back to `apt install texlive-latex-extra texlive-fonts-extra texlive-science`. |
| Final video must include narration AND the scene's animation timing should auto-stretch to match each narration line | `manim-voiceover` | `with self.voiceover(text=...) as tracker:` context manager that exposes `tracker.duration` for `run_time=` and auto-writes SRT subtitles | **Backend must be fixed to local audio files pre-generated by the Manus `generate_speech` tool** (use a `BaseService` subclass that reads an existing WAV/MP3 by filename, or write a 3-line custom service). Never enable the Azure / ElevenLabs / gTTS / CoquiTTS backends; TTS always belongs to `generate_speech`. Install only the upstream `manim-voiceover`; the forks `-fixed` / `-plus` / `-ai` / `-qwen3-tts` are banned because they share the `manim_voiceover` module name and collide. Skip this plugin when the scene has no narration, or when the user only needs a hardcoded narration track without animation-duration coupling (ffmpeg mux in stage 7 is cheaper). |
| Data structures, sorting, graph algorithms, LeetCode-style walkthroughs | `manim-dsa` | `Array`, `LinkedList`, `Stack`, `Queue`, `Tree`, `Graph` Mobjects plus built-in sort/BFS/DFS animations | If the graph has weighted edges, also install the accessory `manim-weighted-line`. |
| Line-by-line code reveal, typewriter effect, code-to-video explainer (Chinese or English) | `manim-code-anim` | Syntax-highlighted code blocks with typewriter reveal and line-highlight cursor | On pure-English code with rendering glitches, fall back to `manim-code-blocks-redux`. |
| Neural networks, CNN/RNN/Transformer diagrams, diffusion-model pipelines | `manim-ml` | Prebuilt ML layer Mobjects so you don't hand-draw hundreds of rectangles | If the scene only needs a plain MLP, the lighter `manim-neural-network` is acceptable. |
| Chemistry: molecules, orbitals, reaction mechanisms, Bohr atom | `manim-chemistry` | Molecule builders, orbital shapes, reaction arrows | For equation-heavy scenes (balancing, mhchem-style), also install `chanim`. 3D molecule views require `ThreeDScene`. |
| Economics/finance models: IS-LM, AD-AS, Solow, Phillips, supply & demand | `manim-ec` | Parametric economic curves with equilibrium markers and shift animations | For stock-specific scenes (candlesticks, volume, moving averages), swap to `manim-stock-visualization` instead. |
| Physics teaching animations: rigid body, EM, waves, optics (formula-driven) | `manim-physics` | Pendulums, springs, field lines, wavefronts as parametric Mobjects | When the user demands a real rigid-body engine (collisions, ropes, friction), switch to `manim-pymunk`. Do not install both simultaneously. |
| Mechanical/engineering drawings: gears, gear trains, dimension lines, CAD figures | `manim-gearbox` (+ `manim-CAD_Drawing_Utils` when dimension lines/chamfers are needed) | Involute gears with correct meshing; CAD-style annotations | For circuit diagrams specifically, use `manim-circuit` (simple symbols) or `manim-eng` (full IEEE library) instead. |
| Agent-to-agent pipeline: the task explicitly asks for a Manim MCP sub-service | `manim-mcp` | Wraps Manim as an MCP server for multi-agent orchestration | Only supports ManimGL under the hood; if Stage 1 selected ManimCE, do not use this plugin. |

The installation snippet to place at the top of `video.py` when any plugin is required:

```python
import importlib, subprocess, sys

def _ensure(*pkgs):
    missing = [p for p in pkgs if importlib.util.find_spec(p.replace("-", "_")) is None]
    if missing:
        subprocess.run([sys.executable, "-m", "pip", "install", "-q", *missing], check=True)

_ensure("manim-dsa", "manim-weighted-line")  # example; replace with actual triggers
```

Keep this block compact. One `_ensure(...)` call per `video.py`, not one per scene.

## 2. Do-not-install list

These plugins exist on PyPI but the `/manim` skill must not install them. Each class is listed with the single reason why.

**TTS itself is always Manus-native.** Speech synthesis always goes through the Manus `generate_speech` tool. Do not install `manim-voiceover-fixed`, `manim-voiceover-plus`, `manim-voiceover-ai`, `manim-voiceover-qwen3-tts`, `manim-speech`, `manim-recorder` — they either try to call third-party TTS backends or duplicate functionality already covered by `generate_speech`. The upstream `manim-voiceover` is allowed, **but only in its local-audio-file mode** (see the trigger table); all TTS backend calls inside it must remain disabled.

**Not an animation-generation job.** Do not install `manim-slides`, `manim-pptx`, `manim-revealjs`, `manim-beamer`, `manim-presentation`, `manim-presentations`, `manim-present`, `manim-editor`. These are downstream packagers (HTML/PPTX/Reveal.js decks). Deck delivery belongs to a slides-oriented skill, not `/manim`.

**Already in Manim's core.** Do not install `manim-table` (Manim CE has `Table`/`MathTable`), `manim-graphing` (archived; replaced by `Axes`/`ParametricFunction`), `manim-progress-bar` (a `Rectangle` + `ValueTracker` is enough and avoids a dependency).

**Experimental or platform-specific.** Do not install `manim-skia` (v0.0.1 experimental renderer), `manim-studio` (GUI), `manim-interactive` (VSCode binding), `manim-livestream` (RTMP), `manim-main-function` (only saves one CLI call), `manim-widget` (Jupyter embedding; leave that to the user's own notebook), `manim-captcha`, `manim-Astronomy`, `manim-lamination-builder`, `manim-rhythm-graph`. They either don't fit agent workflows or are too niche to list here.

**Locale dialects.** Do not install `manim-fa`, `manim-fa-physics`, `manim-devanagari` by default. Only install them if the prompt explicitly requires Persian RTL, Hindi Devanagari, or similar locale-specific typography.

**Miscellaneous and personal toolkits.** Do not install `manim-extras`, `manim-course-utils`, `manim-utils`, `manim-json`, `manim-kodisc`, `manim-tools`, `manim-plugintemplate`, `manim-claude`. They are unmaintained or overlapping personal packages with no reliable contract.

**Deprecated siblings of approved plugins.** Do not install `manim-code-blocks` (use `manim-code-anim` or `-redux`), `manim-chessrender` (use `manim-chess` when needed), `manim-algorithm` / `manim-data-structures` (use `manim-dsa`), `manim-neural-network` (use `manim-ml` unless the scene is a bare MLP), any `manim-voiceover-*` fork (use upstream `manim-voiceover` only).

## 3. Accessory plugins (only when the trigger table routes to them)

The following plugins are not primary scene drivers but may be installed as accessories when a primary trigger already matched. Each accessory has a fixed pairing with a primary plugin.

| Accessory | Pair-with | When to add |
|---|---|---|
| `manim-weighted-line` | `manim-dsa` | Graph has edge weights. |
| `chanim` | `manim-chemistry` | Heavy chemical equations / mhchem-style. |
| `manim-pymunk` | replaces `manim-physics` | Prompt says "real collisions / rope / friction". |
| `manim-stock-visualization` | replaces `manim-ec` | Prompt is about candlesticks, volume, moving averages. |
| `manim-circuit` or `manim-eng` | replaces `manim-gearbox` | Scene is a circuit rather than mechanical. |
| `manim-CAD_Drawing_Utils` | pairs with `manim-gearbox` | Dimension lines, chamfers, fillets, section marks. |
| `manim-code-blocks-redux` | replaces `manim-code-anim` | Pure-English code rendered with glitches. |
| `manim-sequence-diagram` | standalone (trigger: HTTPS / RPC / microservice calls) | UML-style vertical sequence diagrams. |
| `manim-timeline` | standalone (trigger: literature review, roadmap, milestones) | Horizontal scrolling timeline Mobject. |
| `manim-gearbox` without `-CAD` | `manim-physics` replacement in mechanical scenes | Pure gear train visualizations. |
| `manim-euclid` | standalone (trigger: classical Euclidean geometry constructions) | Circumcircle, orthocenter, Euler line, etc. |
| `manim-chess` / `manim-rubikscube` | standalone (trigger: chess puzzle / Rubik's cube tutorial) | Full 8×8 or 3×3×3 board Mobject. |
| `manim-fa` / `manim-devanagari` | standalone (trigger: explicit locale request) | Persian RTL / Hindi Devanagari shaping. |
| `manim-tikz` | standalone (trigger: user supplies existing TikZ source) | Compile TikZ to SVGMobject. |

If none of the trigger table or accessory patterns match, do not install any plugin; write the scene with Manim's core Mobjects.
