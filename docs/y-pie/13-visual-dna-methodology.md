# Visual DNA Methodology

## Purpose

Define Visual DNA as an explainable, evolving personal visual-language model rather than a novelty score or a single opaque embedding.

## Definition

Visual DNA is a versioned representation of recurring, preferred, rejected and evolving visual traits across a selected corpus and time window.

It may describe:

- the user's authored work;
- collected inspiration;
- personal memories;
- a specific project;
- a defined creative period.

These corpora must never be mixed silently.

## Core principles

1. **Corpus-relative** — traits are meaningful relative to declared data.
2. **Time-aware** — the model preserves evolution and rupture.
3. **Multidimensional** — no single `Yannick score` is authoritative.
4. **Explainable** — outputs cite examples and counterexamples.
5. **Feedback-calibrated** — explicit choices outweigh passive inference.
6. **Uncertainty-preserving** — hypotheses remain hypotheses.
7. **Context-separated** — personal memory and artistic authorship are distinct contexts.

## Visual DNA dimensions

### Palette language

- dominant hue families;
- warm/cool distribution;
- saturation profile;
- luminance profile;
- contrast strategy;
- recurring color pairings;
- project-specific palettes;
- palette evolution.

### Composition language

- symmetry and asymmetry;
- centering;
- negative space;
- density and complexity;
- horizon placement;
- depth and layering;
- geometric rhythm;
- focal hierarchy;
- preferred aspect ratios.

### Form and geometry

- organic versus rectilinear forms;
- spirals, circles, grids, arches and radial structures;
- repetition and fractality;
- enclosure and openness;
- verticality and horizontality;
- human-scale versus monumental scale.

### Material and texture

- stone, wood, water, vegetation, glass, metal and light;
- smooth versus textured surfaces;
- natural versus synthetic appearance;
- atmospheric haze, glow and shadow;
- tactile density.

### Semantic vocabulary

- architecture;
- nature;
- technology;
- people;
- animals;
- sacred or ceremonial space;
- domestic intimacy;
- cosmic or civilizational imagery;
- transformation and passage.

### Symbolic and archetypal hypotheses

- sanctuary;
- threshold;
- ascent;
- center;
- bridge;
- garden;
- vessel;
- network;
- union of nature and technology.

These are interpretive hypotheses and require evidence across multiple assets and contexts.

### Emotional register

- serenity;
- awe;
- intimacy;
- melancholy;
- tension;
- transcendence;
- playfulness;
- futurity;
- nostalgia.

### Creative behavior

- tendency toward series;
- exploration versus refinement;
- recurrence frequency;
- abandonment and return;
- preference for hero images or ecosystems;
- variation density;
- acceptance/rejection patterns.

## Model layers

### Layer 1 — Trait observations

Per-asset measurable or inferred traits.

### Layer 2 — Corpus distributions

Frequencies, clusters, co-occurrences and temporal distributions.

### Layer 3 — Stable signatures

Traits persistent across multiple periods and projects.

### Layer 4 — Contextual signatures

Traits specific to a project, place, role or medium.

### Layer 5 — Evolution model

Emergence, strengthening, weakening, rupture, return and hybridization.

### Layer 6 — Narrative interpretation

Human-readable synthesis grounded in representative evidence.

## Creative periods

A period is not created merely because a cluster exists.

Candidate period criteria:

- minimum duration or meaningful bounded event;
- coherent change in several dimensions;
- sufficient sample size;
- distinction from adjacent periods;
- interpretable representative assets;
- optional human confirmation.

Allowed change types:

- emergence;
- intensification;
- simplification;
- rupture;
- return;
- synthesis;
- project migration.

## Originality

Originality must be separated into:

1. `intra_library_distinctiveness` — rarity inside the personal corpus;
2. `series_distinctiveness` — novelty within a series;
3. `project_distinctiveness` — novelty inside a project language;
4. `reference_corpus_distinctiveness` — difference from an explicitly named external corpus;
5. `conceptual_originality_hypothesis` — model interpretation, never objective fact.

## Personal alignment

Y-PIE may estimate alignment to a Visual DNA profile, but must output dimensions and reasons rather than only a percentage.

Example:

```yaml
alignment:
  overall_similarity: 0.82
  profile: elysium-v3
  supporting_traits:
    - organic architecture
    - mineral palette
    - central luminous threshold
  conflicting_traits:
    - high visual clutter
    - commercial typography
  confidence: 0.71
```

## Feedback hierarchy

Evidence priority:

1. explicit user correction or declaration;
2. repeated user selection under a known task;
3. curated collection membership;
4. favorites and edits;
5. passive behavioral signals;
6. model-only inference.

Passive signals never override explicit feedback.

## Hidden-gem definition

A hidden gem is not simply an aesthetically high-scoring image.

Candidate conditions:

- high value on one or more declared dimensions;
- low prior visibility or selection;
- low redundancy;
- meaningful project or corpus fit;
- sufficient confidence;
- no evidence of explicit rejection.

## Evaluation

Visual DNA quality is tested through:

- retrieval: can it find assets the user considers representative?
- discrimination: can it distinguish project languages?
- explanation: are cited traits understandable and grounded?
- stability: do small corpus changes avoid arbitrary profile shifts?
- evolution sensitivity: does it detect known changes without inventing periods?
- feedback adaptation: do corrections change future rankings appropriately?

## MVP relationship

Visual DNA is not an MVP dependency.

The MVP must only preserve the data required for future Visual DNA:

- versioned embeddings;
- technical and semantic observations;
- timestamps;
- collection context;
- explicit feedback;
- source provenance;
- model versions.

No Visual DNA dashboard is required before the core pipeline is reliable.
